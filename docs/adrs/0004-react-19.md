# ADR 0004: React 19

Datum: 2026-07-12 · Status: Akzeptiert

## Kontext

React 19 bringt fundamentale Neuerungen: Server Components als Default,
Actions, `use()`-Hook, verbesserte Suspense-Semantik und optimierte
Hydration. Das Blog nutzt Next.js 16, das React 19 voraussetzt.

## Entscheidung

Wir verwenden React 19 (aktuell 19.2.3) über alle Packages hinweg.

## Begründung

- **Server Components**: Blog-Posts können als Server Components gerendert
  werden — kein Client-JS für statischen Content.
- **Reduktion der Bundle Size**: Weniger JavaScript beim Client durch
  serverseitiges Rendering von Content-Komponenten.
- **Next.js 16 Requirement**: Next.js 16 setzt React 19 als Minimum voraus.
- **Zukunftssicherheit**: React 19 ist die aktuelle Stable-Linie; Libraries
  im Ecosystem werden darauf ausgerichtet.

## Konsequenzen

- Komponenten sind standardmäßig Server Components; `'use client'` wird
  explizit nur dort gesetzt, wo Interaktivität gebraucht wird.
- Peer-Dependencies in der Panorama-Library fordern `react >= 19.2.3`.
- Legacy-Patterns (Class Components, `componentDidMount`) werden nicht
  verwendet.
