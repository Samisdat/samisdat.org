# Security Policy

## Supported Versions

This is a personal blog project. Security updates are applied to the `main` branch only.

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

### Preferred Method: Private Vulnerability Reporting

Use GitHub's [Private Vulnerability Reporting](https://github.com/Samisdat/samisdat.org/security/advisories/new) feature (Security tab → "Report a vulnerability"). This creates a private advisory that allows us to discuss and fix the issue before public disclosure.

### Alternative: Email

If you prefer email or if private reporting is unavailable, send your report to:

**bastian@pertz.eu**

Please include:

- A description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: You'll receive a response within 72 hours confirming receipt.
- **Assessment**: We'll evaluate the vulnerability and determine severity.
- **Fix Timeline**: For confirmed vulnerabilities:
  - Critical/High: Fix within 7 days
  - Medium: Fix within 30 days
  - Low: Fix in the next regular release cycle
- **Disclosure**: Once a fix is available and deployed, we'll coordinate public disclosure with you.

### Out of Scope

This project does not offer a bug bounty program. The following are explicitly out of scope:

- Vulnerabilities in dependencies (report directly to the upstream project or wait for Dependabot)
- Issues affecting only old browsers not listed in our browserslist configuration
- Social engineering attacks

Thank you for helping keep this project secure!
